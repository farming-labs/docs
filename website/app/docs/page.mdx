# @farming-labs/docs

A modern documentation framework built on top of [Fumadocs](https://fumadocs.vercel.app). One config file, zero boilerplate, beautiful themes out of the box.

## Why @farming-labs/docs?

- **Zero boilerplate** — No layout files, no `[[...slug]]` wrappers. Just write Markdown or MDX.
- **One config** — Everything in `docs.config.ts`: theme, colors, typography, icons, components, metadata.
- **Multiple frameworks** — First-class support for both Next.js and SvelteKit.
- **Three themes** — Default, Darksharp, and Pixel Border. Or build your own with `createTheme()`.
- **Built-in search** — Full-text search powered by Orama. No API keys.
- **CLI scaffolding** — `npx @farming-labs/docs init` detects your framework, generates config, installs deps, starts dev.

## Packages

| Package | Description |
|---------|-------------|
| `@farming-labs/docs` | Core types, `defineDocs()`, `createTheme()`, `extendTheme()` |
| `@farming-labs/theme` | Next.js theme presets, layout components, CSS presets |
| `@farming-labs/next` | Next.js adapter — `withDocs()`, MDX processing, search API |
| `@farming-labs/svelte` | SvelteKit adapter — server-side docs loader, markdown processing |
| `@farming-labs/svelte-theme` | SvelteKit theme presets, DocsLayout, DocsContent components |

## Quick Start

```bash
npx @farming-labs/docs init
```

The CLI auto-detects your framework (Next.js or SvelteKit) and scaffolds everything you need.

Or install manually — see the [Installation](/docs/installation) guide.
