# CLI

The `@farming-labs/docs` CLI handles scaffolding, upgrades, health checks, cloud deploys, agent compaction, search sync, and more. All commands are available via `npx @farming-labs/docs@latest` or through your local install with `pnpm exec docs`.

## Commands

| Command | Description |
|---|---|
| `init` | Scaffold docs in your project. Detects framework, installs packages, creates `docs.config.ts`, and generates starter pages. |
| `dev` | Run frameworkless docs locally from `docs.json`. |
| `deploy` | Sync cloud config and deploy hosted preview docs. |
| `preview` | Alias for `deploy`. |
| `cloud` | Docs Cloud utilities: `init`, `check`, `deploy`, `preview`, `sync`. |
| `agent` | Agent utilities: `compact` generates sibling `agent.md` files. |
| `doctor` | Health-check your docs site for agent-readiness or reader-facing quality. |
| `upgrade` | Upgrade all docs packages to the latest (or a specific) release. |
| `robots` | Generate or update `robots.txt` with docs-aware directives. |
| `sitemap` | Generate `sitemap.xml`, `sitemap.md`, and `docs/sitemap.md`. |
| `search` | Sync the search index with Algolia or Typesense. |
| `mcp` | Run the MCP server over stdio. |
| `review` | Run a docs review against changed files (used in CI). |
| `codeblocks` | Validate code blocks in your MDX files. |

## `docs doctor` — key options

`docs doctor` is the primary way to validate your docs site before shipping. It supports two audit modes and several flags for CI integration:

```bash
# Agent-readiness audit (default)
pnpm exec docs doctor

# Reader-facing quality audit
pnpm exec docs doctor --site

# Fail CI on warnings, output JSON
pnpm exec docs doctor --agent --strict --json

# Fail CI only on hard failures, tolerate warnings
pnpm exec docs doctor --agent --fail-on fail
```

The `--fail-on` flag accepts `warn` or `fail`:

- `warn` — exit non-zero on any warning or failure (same as `--strict`).
- `fail` — exit non-zero only on hard failures; warnings are tolerated.

For the complete flag reference, see [CLI usage](/docs/api/cli/usage).
