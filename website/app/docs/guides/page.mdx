## What These Guides Optimize For

- **One source of truth** - the human page stays canonical unless a route truly needs a dedicated `agent.md`
- **Better page contracts** - clear descriptions, related pages, verification steps, and failure states
- **Useful machine context** - additive `Agent` blocks and machine-readable routes instead of HTML scraping
- **Operational docs** - pages that help an agent complete a task, not just describe a feature

<Callout type="info" title="Guides are just docs pages">
  Put them under <code>app/docs/guides/.../page.mdx</code> and they automatically inherit the docs
  layout, <code>.md</code> routes, search, MCP readability, and page actions. You do not need a
  separate blog engine or a parallel content system for long-form writing.
</Callout>
