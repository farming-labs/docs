# CLI

The `@farming-labs/docs` CLI has two main jobs:

- **`init`** тАФ create a new docs app or add docs to an existing app
- **`upgrade`** тАФ bump the docs packages in an existing project

If you only need the common commands, start here. The framework-specific output and generated file lists are further down this page.

## Quick Start

### From scratch

Use `init --template` when you want a brand new project directory with docs already set up.

<Tabs items={["npm", "pnpm", "yarn", "bun"]}>

## Init

### `init` without `--template`

When you run `init` **without** `--template`, the CLI asks whether you're working with an **existing project** or starting **from scratch**.

| Choice | Best for | What happens |
| ------ | -------- | ------------ |
| `Existing project` | You already have a Next.js, TanStack Start, Nuxt, SvelteKit, or Astro app | The CLI adds docs to the current app |
| `Fresh project` | You want a new docs starter | The CLI creates a new folder from the selected example template |

### From scratch

If you select **Fresh project**, the CLI asks for:

1. **Framework** тАФ Next.js, TanStack Start, Nuxt, SvelteKit, or Astro
2. **Project name** тАФ default is `my-docs`
3. **Package manager** тАФ used for install and the suggested dev command

You can skip those prompts entirely with `--template` and `--name`.

<Tabs items={["npm", "pnpm", "yarn", "bun"]}>
  <Tab value="npm">
    ```bash
    npx @farming-labs/docs init --template next --name my-docs
    ```
  </Tab>
  <Tab value="pnpm">
    ```bash
    pnpm dlx @farming-labs/docs init --template next --name my-docs
    ```
  </Tab>
  <Tab value="yarn">
    ```bash
    yarn dlx @farming-labs/docs init --template next --name my-docs
    ```
  </Tab>
  <Tab value="bun">
    ```bash
    bunx @farming-labs/docs init --template next --name my-docs
    ```
  </Tab>
</Tabs>

Replace `next` with `tanstack-start`, `nuxt`, `sveltekit`, or `astro`.

### Existing project

If you select **Existing project**, the CLI stays in the current folder and wires docs into the app you already have.

1. **Framework** тАФ Auto-detected from `package.json`, or you pick one if detection fails.
2. **Theme** тАФ One of the built-in themes or **Create your own theme** (youтАЩll be asked for a theme name; it creates `themes/<name>.ts` and `themes/<name>.css`).
3. **Path aliases** тАФ Whether to use `@/`, `$lib/`, or `~/` for imports (framework-dependent).
4. **Entry path** тАФ Where docs live (e.g. `docs`). Press **Enter** to use the default.
5. **Optional API reference** тАФ Whether to scaffold API reference support. If you enable it, the CLI pre-fills the detected API route root (usually `api`) and lets you edit it before writing `apiReference.routeRoot`.
6. **Optional i18n** тАФ Whether to scaffold internationalized docs, which locales to generate, any extra locale codes to add, and which locale should be the default.
7. **Global CSS** тАФ Which file to use (auto-detected if thereтАЩs only one, otherwise you pick or type the path; Enter uses the default).
8. The CLI generates config, layout, and sample pages, then detects the package manager from a lockfile when possible. If no lockfile is found, it asks which package manager you want to use before installing dependencies, and can then start the dev server.

Typical flow:

<Tabs items={["npm", "pnpm", "yarn", "bun"]}>
  <Tab value="npm">
    ```bash
    cd my-app
    npx @farming-labs/docs init
    ```
  </Tab>
  <Tab value="pnpm">
    ```bash
    cd my-app
    pnpm dlx @farming-labs/docs init
    ```
  </Tab>
  <Tab value="yarn">
    ```bash
    cd my-app
    yarn dlx @farming-labs/docs init
    ```
  </Tab>
  <Tab value="bun">
    ```bash
    cd my-app
    bunx @farming-labs/docs init
    ```
  </Tab>
</Tabs>

Prompts that show a **placeholder** (e.g. entry path `docs`, project name `my-docs`, theme name `my-theme`) use that value as the default тАФ you can press **Enter** to accept it instead of typing.

### Existing-project flags

Use these to skip the matching prompts during `init`:

| Flag | Description |
| ---- | ----------- |
| `--theme <name>` | Skip the theme prompt |
| `--entry <path>` | Skip the docs entry prompt |
| `--api-reference` | Enable API reference scaffold during `init` |
| `--no-api-reference` | Explicitly skip API reference scaffold |
| `--api-route-root <path>` | Set the detected API route root written to `apiReference.routeRoot` |

Example:

```bash
npx @farming-labs/docs init --theme greentree --entry docs --api-reference --api-route-root internal-api
```

`--api-route-root` is optional. If you omit it, the CLI tries to detect a sensible default from the frameworkтАЩs route conventions and shows that value as the default in the prompt.

When API reference is enabled through `init`:

- **Next.js** writes the `apiReference` block and `app/api-reference/[[...slug]]/route.ts` (or `src/app/...`).
- **TanStack Start, SvelteKit, Astro, and Nuxt** write the `apiReference` block and also scaffold the minimal `/{path}` handler files so the reference works immediately.

### Optional i18n scaffold

If you enable i18n during `init`, the CLI will:

1. Let you multi-select common locales such as `en`, `fr`, `es`, `de`, `pt`, `ja`, `ko`, `zh`, `ar`, `hi`, and `ru`
2. Let you add custom locale codes such as `pt-BR`
3. Ask which locale should be the default

The generated project gets:

- An `i18n` block in `docs.config.ts` / `src/lib/docs.config.ts`
- Locale content folders like `docs/en`, `docs/fr`, and `docs/zh`
- Framework-specific starter files so routes like `/docs?lang=en` and `/docs?lang=fr` work out of the box

## Upgrade

Upgrade all `@farming-labs/*` docs packages to the latest version. Run from your project root; the CLI **auto-detects** your framework from `package.json` and upgrades the right packages for Next.js, TanStack Start, Nuxt, SvelteKit, and Astro.

The CLI also detects your package manager from the lockfile in the current directory (`pnpm-lock.yaml`, `yarn.lock`, `bun.lock`, `bun.lockb`, or `package-lock.json`). If no lockfile is found, it asks which package manager you want to use instead of defaulting to npm.

<Tabs items={["npm", "pnpm", "yarn", "bun"]}>
  <Tab value="npm">
    ```bash
    npx @farming-labs/docs upgrade
    ```
  </Tab>
  <Tab value="pnpm">
    ```bash
    pnpm dlx @farming-labs/docs upgrade
    ```
  </Tab>
  <Tab value="yarn">
    ```bash
    yarn dlx @farming-labs/docs upgrade
    ```
  </Tab>
  <Tab value="bun">
    ```bash
    bunx @farming-labs/docs upgrade
    ```
  </Tab>
</Tabs>

| Framework | Packages upgraded to `@latest` |
| --------- | ------------------------------ |
| Next.js   | `@farming-labs/docs`, `@farming-labs/theme`, `@farming-labs/next` |
| TanStack Start | `@farming-labs/docs`, `@farming-labs/theme`, `@farming-labs/tanstack-start` |
| Nuxt      | `@farming-labs/docs`, `@farming-labs/nuxt`, `@farming-labs/nuxt-theme` |

| Astro     | `@farming-labs/docs`, `@farming-labs/astro`, `@farming-labs/astro-theme` |

To specify the framework explicitly (e.g. in a monorepo or when auto-detect fails), use `--framework`:

Valid values: `next`, `tanstack-start`, `nuxt`, `sveltekit`, `astro`.

The init CLI offers several built-in themes plus an option to create your own:

| Theme          | Description                                                |
| -------------- | ---------------------------------------------------------- |
| `fumadocs`     | Clean, neutral docs theme (default)                        |
| `darksharp`    | All-black, sharp corners, no rounded edges                 |
| `pixel-border` | Refined dark UI inspired by better-auth.com                |
| `colorful`     | Fumadocs-style neutral theme with description support      |

| **Create your own theme** | Scaffolds `themes/<name>.ts` and `themes/<name>.css`; youтАЩre prompted for the theme name (default `my-theme`). See [Creating your own theme](/docs/themes/creating-themes). |

## What it does (Existing project)

When you choose **Existing project**, the CLI does the following (framework-specific files vary):

<Tabs items={["Next.js", "TanStack Start", "SvelteKit", "Astro", "Nuxt"]}>
  <Tab value="Next.js">
    1. **Detects Next.js** via `next` in your `package.json` (or asks you to pick a framework)
    2. **Asks for a theme** тАФ built-in themes or **Create your own theme** (then prompts for theme name; press Enter for default `my-theme`)
    3. **Asks for the entry path** тАФ Default is `docs` (press Enter to accept)
    4. **Asks about path aliases** тАФ Use `@/` alias or relative paths for imports
    5. **Optionally scaffolds i18n** тАФ Generates locale folders like `app/docs/en` and `app/docs/fr`, plus locale-aware wrapper pages
    6. **Generates files:**
       - `docs.config.ts` тАФ Full config with your selected theme
       - `next.config.ts` тАФ Wraps your existing config with `withDocs()`
       - `app/global.css` тАФ Imports Tailwind and theme CSS
       - `app/layout.tsx` тАФ Root layout with RootProvider
       - `app/docs/layout.tsx` тАФ Docs layout
       - `app/docs/page.mdx` тАФ Sample documentation page when i18n is disabled
       - `app/docs/installation/page.mdx` тАФ Sample installation guide when i18n is disabled
       - `app/docs/quickstart/page.mdx` тАФ Sample quickstart guide when i18n is disabled
    7. **Installs dependencies** тАФ `@farming-labs/docs`, `@farming-labs/theme`, `@farming-labs/next`
    8. **Starts the dev server** тАФ Opens your docs at `http://localhost:3000/docs`
  </Tab>
  <Tab value="TanStack Start">
    1. **Detects TanStack Start** via `@tanstack/react-start` in your `package.json` (or asks you to pick a framework)
    2. **Asks for a theme** тАФ built-in themes or **Create your own theme** (then prompts for theme name; press Enter for default `my-theme`)
    3. **Asks about path aliases** тАФ Use `@/` alias or relative paths for imports
    4. **Asks for the entry path** тАФ Default is `docs` (press Enter to accept)
    5. **Skips the built-in i18n scaffold** so the generated TanStack routes stay minimal and working
    6. **Generates files:**
       - `docs.config.ts` тАФ Full config with your selected theme
       - `src/lib/docs.server.ts` тАФ TanStack docs server
       - `src/lib/docs.functions.ts` тАФ Server function wrapper for page loading
       - `src/routes/{entry}/index.tsx` тАФ Docs index route
       - `src/routes/{entry}/$.tsx` тАФ Catch-all docs route
       - `src/routes/api/docs.ts` тАФ Search and AI API route
       - `src/styles/app.css` тАФ Imports Tailwind and theme CSS
       - `docs/page.mdx` тАФ Sample documentation page
       - `docs/installation/page.mdx` тАФ Sample installation guide
       - `docs/quickstart/page.mdx` тАФ Sample quickstart guide
    7. **Updates existing app files when needed** тАФ Injects `RootProvider` into `src/routes/__root.tsx` and adds `docsMdx()` / Tailwind plugins to `vite.config.ts`
    8. **Installs dependencies** тАФ `@farming-labs/docs`, `@farming-labs/theme`, `@farming-labs/tanstack-start`
    9. **Starts the dev server** тАФ Opens your docs at `http://localhost:5173/docs`
  </Tab>
  <Tab value="SvelteKit">
    1. **Detects SvelteKit** via `@sveltejs/kit` in your `package.json` (or asks you to pick a framework)
    2. **Asks for a theme** тАФ built-in themes or **Create your own theme** (then prompts for theme name; press Enter for default)
    3. **Asks for the entry path** тАФ Default is `docs` (press Enter to accept)
    4. **Asks about path aliases** тАФ Use `$lib/` alias or relative paths for imports
    5. **Optionally scaffolds i18n** тАФ Generates locale folders like `docs/en` and `docs/fr` plus a docs root page for `/docs?lang=...`
    6. **Generates files (does not touch `svelte.config.js`):**
       - `src/lib/docs.config.ts` тАФ Full config with your selected theme
       - `src/lib/docs.server.ts` тАФ Server-side docs loader with `import.meta.glob` for production builds
       - `src/routes/docs/+layout.svelte` тАФ Docs layout component
       - `src/routes/docs/+layout.server.js` тАФ Layout data loader
       - `src/routes/docs/[...slug]/+page.svelte` тАФ Dynamic page component
       - `src/routes/docs/+page.svelte` тАФ Generated when i18n is enabled
       - `src/app.css` тАФ Theme CSS import
       - `docs/page.md` тАФ Sample documentation page
       - `docs/installation/page.md` тАФ Sample installation guide
       - `docs/quickstart/page.md` тАФ Sample quickstart guide
    7. **Installs dependencies** тАФ `@farming-labs/docs`, `@farming-labs/svelte`, `@farming-labs/svelte-theme`
    8. **Starts the dev server** тАФ Opens your docs at `http://localhost:5173/docs`

    > **Production note**: The generated `docs.server.ts` uses `import.meta.glob` to bundle your markdown files at build time. This is required for serverless deployments (Vercel, Netlify, etc.) where the filesystem is not available at runtime.

  </Tab>
  <Tab value="Astro">
    1. **Detects Astro** via `astro` in your `package.json` (or asks you to pick a framework)
    2. **Asks for a theme** тАФ built-in themes or **Create your own theme** (then prompts for theme name; press Enter for default)
    3. **Asks for the entry path** тАФ Default is `docs` (press Enter to accept)
    4. **Optionally scaffolds i18n** тАФ Generates locale folders like `docs/en` and `docs/fr`
    5. **Generates files:**
       - `src/lib/docs.config.ts` тАФ Full config with your selected theme
       - `src/lib/docs.server.ts` тАФ Server-side docs loader with `import.meta.glob` for production builds
       - `src/pages/{entry}/index.astro` тАФ Docs index page
       - `src/pages/{entry}/[...slug].astro` тАФ Dynamic page component
       - `src/pages/api/{entry}.ts` тАФ API route for search and AI
       - `astro.config.mjs` тАФ Astro config with SSR enabled
       - `docs/page.md` тАФ Sample documentation page
       - `docs/installation/page.md` тАФ Sample installation guide
       - `docs/quickstart/page.md` тАФ Sample quickstart guide
    6. **Installs dependencies** тАФ `@farming-labs/docs`, `@farming-labs/astro`, `@farming-labs/astro-theme`
    7. **Starts the dev server** тАФ Opens your docs at `http://localhost:4321/docs`

    > **Production note**: The generated `docs.server.ts` uses `import.meta.glob` to bundle your markdown files at build time. This is required for serverless deployments (Vercel, Netlify, etc.) where the filesystem is not available at runtime.

  </Tab>
  <Tab value="Nuxt">
    1. **Detects Nuxt** via `nuxt` in your `package.json` (or asks you to pick a framework)
    2. **Asks for a theme** тАФ built-in themes or **Create your own theme** (then prompts for theme name; press Enter for default)
    3. **Asks for the entry path** тАФ Default is `docs` (press Enter to accept)
    4. **Optionally scaffolds i18n** тАФ Generates locale folders like `docs/en` and `docs/fr`
    5. **Generates files:**
       - `docs.config.ts` тАФ Full config with your selected theme
       - `nuxt.config.ts` тАФ Nuxt config with theme CSS and server assets
       - `server/api/docs.get.ts`, `server/api/docs.post.ts`, `server/api/docs/load.get.ts` тАФ docs data, search, AI, and page loading
       - `pages/docs/[[...slug]].vue` тАФ Catch-all page component that handles the docs index and nested docs routes
       - `docs/page.md` тАФ Sample documentation page
       - `docs/installation/page.md` тАФ Sample installation guide
       - `docs/quickstart/page.md` тАФ Sample quickstart guide
    6. **Installs dependencies** тАФ `@farming-labs/docs`, `@farming-labs/nuxt`, `@farming-labs/nuxt-theme`
    7. **Starts the dev server** тАФ Opens your docs at `http://localhost:3000/docs`
  </Tab>
</Tabs>

## Flags

| Flag                    | Description                                                                 |
| ----------------------- | --------------------------------------------------------------------------- |
| `--template <name>`     | Bootstrap a project: `next`, `tanstack-start`, `nuxt`, `sveltekit`, or `astro`. Use with `--name`. |
| `--name <project>`      | Project folder name when using `--template`; prompt if omitted (e.g. `my-docs`) |
| `--theme <name>`        | Skip theme prompt: `fumadocs`, `darksharp`, `pixel-border`, `colorful`, `darkbold`, `shiny`, `greentree` |
| `--entry <path>`        | Skip entry path prompt (e.g. `--entry docs`)                                |

**Starting from scratch?** Use `--template` with `--name <project>`. The CLI bootstraps a project with the given name, installs dependencies, and then you run `cd <project> && pnpm run dev` to start.

<Callout type="info" title="Theme CSS in global styles">
  For each framework, add your **theme's CSS** to the global stylesheet (or config) shown below. The import must match the theme you use in `docs.config` (e.g. `default`, `greentree`, `darksharp`). Without it, docs pages will lack the correct styling.
</Callout>

<Callout type="info" title="Rename to docs.config.ts when using JSX">
  If your config includes JSX or React nodes, rename `docs.config.ts` to `docs.config.ts`.
  This applies when you add things like a custom `nav.title`, React icons, or React-based components directly in the config file.
</Callout>
