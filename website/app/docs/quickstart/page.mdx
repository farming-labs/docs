# Quickstart

Get a working docs site in under two minutes.

## Add to an existing app

Run the init command inside your Next.js, TanStack Start, SvelteKit, Astro, or Nuxt project:

```bash title="terminal"
npx @farming-labs/docs init
```

The CLI detects your framework, installs the right packages, creates `docs.config.ts`, adds the theme CSS, scaffolds starter pages, and starts the dev server.

## Start a new project

Use `--template` with `--name` to bootstrap a ready-to-run project without prompts:

<CodeGroup>
```bash title="Next.js"
npx @farming-labs/docs init --template next --name my-docs
```
```bash title="TanStack Start"
npx @farming-labs/docs init --template tanstack-start --name my-docs
```
```bash title="Nuxt"
npx @farming-labs/docs init --template nuxt --name my-docs
```
```bash title="SvelteKit"
npx @farming-labs/docs init --template sveltekit --name my-docs
```
```bash title="Astro"
npx @farming-labs/docs init --template astro --name my-docs
```
</CodeGroup>

The CLI creates the project folder, installs dependencies, and exits. No interactive prompts when `--template` and `--name` are both provided.

## What gets created

After init, your project has:

- `docs.config.ts` — single config file for theme, search, metadata, AI, and agent surfaces
- A theme CSS import in your global stylesheet
- A framework-specific docs layout and route file
- A `docs/` folder with a sample `index.mdx` page
