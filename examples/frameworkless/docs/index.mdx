# Frameworkless Docs

This project only contains markdown, `docs.json`, and an OpenAPI source.

`content.openapi[].path` can point to either:

- a local file like `api/openapi.yaml`
- a remote URL like `https://example.com/openapi.json`

This example is currently using a remote OpenAPI URL for `/api-reference`.

## Config Shape

In this example, `docs.json` separates the docs project from the hosted cloud layer:

```json
{
  "docs": {
    "mode": "frameworkless",
    "runtime": "nextjs",
    "root": ".docs/site"
  },
  "cloud": {
    "apiKey": {
      "env": "DOCS_CLOUD_API_KEY"
    },
    "deploy": {
      "enabled": false
    },
    "analytics": {
      "enabled": true,
      "console": "info",
      "includeInputs": false
    }
  }
}
```

- `docs.mode` describes how the project is authored
- `docs.runtime` is the generated docs runtime choice
- `cloud.apiKey.env` tells CLI/CI where to read the Docs Cloud API key without writing secrets to JSON
- `cloud.deploy.enabled` controls whether `docs deploy` can create hosted preview docs
- `cloud.analytics` is the Docs Cloud analytics contract for the serializable runtime subset

Even with `cloud.deploy.enabled: false`, frameworkless local preview can still materialize the
serializable `cloud.analytics` subset into the generated hidden runtime.

See the full contract in the
[`docs.json` guide](https://docs.farming-labs.dev/docs/guides/docs-json).

Docs Cloud itself is currently waitlist-based. If you want preview deploys, publishing workflows,
AI, or managed hosting, join the waitlist on the
[Cloud page](https://docs.farming-labs.dev/cloud).

`docs.runtime` is intentionally explicit in the shared contract so the runtime can be something
other than `nextjs` too. This example is using `nextjs` today.
