# CLI Reference

The `farming-docs` CLI lets you scaffold a new documentation project in seconds.

## Quick Start

```bash
npx farming-docs init
```

The CLI will:

1. Auto-detect your framework (Next.js, Vite, etc.)
2. Let you choose a theme (Fumadocs, custom, etc.)
3. Generate all configuration files
4. Install dependencies
5. Start the dev server

## Commands

### `init`

Initialize a new docs project in the current directory.

```bash
npx farming-docs init [options]
```

| Option | Description | Default |
|--------|-------------|---------|
| `--framework` | Target framework | auto-detect |
| `--theme` | Theme preset | `fumadocs` |
| `--entry` | Docs entry folder | `docs` |

### `dev`

Start the development server.

```bash
npx farming-docs dev
```

### `build`

Build the documentation site for production.

```bash
npx farming-docs build
```

## Examples

### Scaffold with Fumadocs theme

```bash
npx farming-docs init --theme fumadocs --entry docs
```

### Custom entry path

```bash
npx farming-docs init --entry guides
```

This creates pages under `app/guides/` instead of `app/docs/`.
